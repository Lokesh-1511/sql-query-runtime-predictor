# Machine Learning Based SQL Query Runtime Prediction System

## Updated Complete Development Roadmap

This roadmap follows a modular ML pipeline architecture:

* Python scripts handle **data generation and feature extraction**
* Jupyter notebooks handle **EDA, model training, and explainability**
* FastAPI provides **runtime prediction API**
* React frontend (optional) provides **user interface**

Phases 1–3 are completed.

# PHASE 4 — Automated Query Execution Pipeline

Goal: Execute SQL queries repeatedly and record runtime metrics.

Tasks:

1. Implement automated query runner

File:

```
runner/query_runner.py
```

Responsibilities:

* connect to DuckDB
* load query files
* execute queries
* measure runtime

2. Execute each query multiple times (recommended 5–10 runs).

3. Capture runtime using Python `time` module.

4. Save results to dataset:

```
data/query_runtime_logs.csv
```

Columns:

```
query_id
query_text
execution_time
run_number
```
